1. **어노테이션 도구가 v2로 업데이트**되었습니다 — 배포 URL 동일. 새로고침만 하시면 새 UI가 뜹니다.

## 2. 왜 재라벨링하는가

세션 6~7 동안 수행된 v1 파일럿 결과 분석에서 다음이 확인되었습니다.

- **Fleiss κ = 0.315** (박정진·이종훈 간, 7개 교집합 샘플 기준) → Landis & Koch(1977) 기준 **"fair"** 수준으로 본 라벨링 진입 기준 κ ≥ 0.41에 미달
- v1 코딩북은 **단일 주 SDG** 라벨만 허용 → 교육·복지·환경이 얽힌 사업에서 어노테이터 간 합의 곤란
- 해결책으로 코딩북 v1.6에서 **주목표 + 연계목표 이원화**(OECD Rio Markers의 Principal/Significant 방식)와 **Nexus 체크리스트**를 도입

또한 표본 추출 방식이 층화 비율 조정(대도시형 15 · 도시형 25 · 농촌형 10 = 30/50/20)을 반영하도록 재설계되었습니다(옵션 C). 새 50건은 기존 v1과 일부(7건)만 교집합하며 대부분 처음 보시는 사업입니다.

**v1 응답은 아카이브로 보존**되어 있으므로 손실 없음. v1 vs v2 비교 분석에 활용 예정입니다.

## 4. 새 UI 사용법 (1장 요약)

### 4.1 화면 구성 변경

| 구분 | v1 | **v2 (현재)** |
|---|---|---|
| SDG 선택 | 17개 병렬 버튼 | **5P 계층** → SDG 2단계 선택 |
| 라벨 구조 | 주 SDG + 연계 최대 3개 (tier=애매함일 때만) | **주목표 1개 + 연계목표 0~2개** (tier 무관 독립 섹션) |
| Nexus 가이드 | 없음 | **3문항 체크리스트** 상시 노출 |
| 확신도 | 3-tier (확실/애매/모름) | 유지 |
| Red Tag | 4종 (없음/A/B/C) | 유지 (SDG 상충 · SDG Washing · Anti-SDG) |
| 수혜자 구분 | 없음 | **M-01 수혜자/제공자 체크리스트** 신설 |

### 4.2 작업 순서 (권장)

1. **라벨링 체크리스트(좌측)** — "실제 수혜자"와 "공급자 인식 여부" 먼저 확인 → NA-OUTSIDE 오분류 방지
2. **① 주목표 SDG · Principal** — 5P(People/Prosperity/Planet/Peace/Partnership) 1개 선택 → SDG 17개 중 1개 선택 (**주목표는 반드시 1개**)
3. **② 주목표 확신도** — 확실함 / 애매함 / 모르겠음 중 1개
4. **③ 연계목표 SDGs · Significant** — **Nexus 체크리스트 Q1~Q3을 먼저 읽고**, 1개 이상 YES일 때만 추가 (0~2개). 모두 NO이면 연계목표 없이 저장
5. **④ Red Tag** — 해당 시 선택
6. **⑤ 판단 근거** — 한 줄 메모(선택). 애매하거나 Red Tag C(Anti-SDG) 선택 시 권장
7. **저장 후 다음 → / 임시 저장 →**

### 4.3 Nexus 체크리스트 (연계목표 추가 판단 기준)

연계목표 섹션 상단에 상시 노출됩니다. **"혹시 모르니 추가"는 모델 학습 오류를 유발**하므로 반드시 체크리스트 기준대로만 추가하세요.

- **Q1**: 사업이 2개 이상 SDG 영역(People/Prosperity/Planet/Peace/Partnership)에 걸쳐 있는가?
- **Q2**: 사업 수단(인프라·훈련·플랫폼 등)이 주목표 외 다른 SDG에 **기여 또는 저해**하는가?
- **Q3**: 수혜자가 다른 SDG 영역(예: 고령+여성, 환경+산업)에 중복되는가?

→ 1개 이상 YES이면 연계목표 추가. 모두 NO이면 추가 안 함.

## 5. 규칙 (Blind 원칙)

**IAA(어노테이터 간 신뢰도) 산출을 위해 어노테이터끼리 다음 행위를 금합니다**:

- ❌ 서로의 라벨 결과 공유 (완료 전·중·후 모두)
- ❌ 특정 샘플의 "정답"에 대한 논의
- ❌ "이 사업은 SDG 몇 같은데?"라는 상호 문의
- ✅ **코딩북 해석·애매한 정의에 대한 질문**은 박정진에게 1:1로 해주세요 (카톡/이메일/미팅)
- ✅ Red Tag C(Anti-SDG) 선택 시 판단 근거를 **반드시 기록**

애매한 샘플은 "애매함(ambiguous)" 또는 "모르겠음(unknown)"으로 솔직하게 표시해 주세요. 모르는 것을 "확실함"으로 찍는 것이 Blind κ를 가장 크게 훼손합니다.
